# Vekus 钣金AI报价平台 — 服务器部署指南

## 服务器配置评估

| 项目 | 配置 | 评估 |
|------|------|------|
| CPU | 4核 | ✅ 充足，可跑4个 uvicorn worker |
| 内存 | 8GB | ✅ 充足，系统约1GB + 应用约2GB + 余量5GB |
| 系统盘 | 40GB | ✅ 系统约10GB + 应用约1GB，余量充足 |
| 数据盘 | 200GB | ✅ 用于存储上传图纸和数据库，可存数十万图纸 |
| 带宽 | 5Mbps | ✅ 约625KB/s，支持50-100并发用户正常使用 |

**结论：这台S6 4核8G云主机完全满足Vekus平台的运行需求。**

## 部署步骤

### 前置准备

1. 安装系统依赖（CentOS）：
```bash
yum install -y python311 python311-pip nginx git
```

2. 创建应用目录：
```bash
mkdir -p /data/www/vekus/storage/uploads
mkdir -p /data/www/vekus/logs
```

### 后端部署

3. 上传后端代码到 `/data/www/vekus/backend/`

4. 创建 Python 虚拟环境：
```bash
python3.11 -m venv /data/www/vekus/venv
source /data/www/vekus/venv/bin/activate
pip install fastapi uvicorn[standard] sqlalchemy pydantic python-multipart requests
```

5. 创建环境配置文件 `/data/www/vekus/.env`：
```bash
VEKUS_AI_PROVIDER=deepseek
VEKUS_AI_API_KEY=<你的DeepSeek API Key>
AI_CHAT_API_KEY=<你的DeepSeek API Key>
AI_CHAT_API_BASE=https://api.deepseek.com/v1
AI_CHAT_MODEL=deepseek-chat
```

6. 测试后端启动：
```bash
source /data/www/vekus/venv/bin/activate
cd /data/www/vekus
python -m uvicorn backend.app.main:app --host 0.0.0.0 --port 8000
# 另开终端验证: curl http://localhost:8000/api/health
```

### 前端部署

7. 在本地构建前端：
```bash
npm run build
```

8. 上传 `dist/` 目录到 `/data/www/vekus/dist/`

### Nginx 配置

9. 创建 `/etc/nginx/conf.d/vekus.conf`，内容见 `deploy/nginx.conf`

10. 验证并重启 Nginx：
```bash
nginx -t
systemctl reload nginx
```

### 后台服务

11. 创建 systemd 服务 `/etc/systemd/system/vekus-api.service`，内容见 `deploy/vekus-api.service`

12. 启动服务：
```bash
systemctl daemon-reload
systemctl enable vekus-api
systemctl start vekus-api
systemctl status vekus-api
```

### 验证

13. 检查服务状态：
```bash
curl http://localhost/health          # Nginx -> API 代理
curl http://localhost/api/health       # API 健康检查
curl http://localhost/                 # 前端页面
```

## 日常运维

```bash
# 查看服务状态
systemctl status vekus-api

# 重启服务
systemctl restart vekus-api

# 查看日志
journalctl -u vekus-api -f

# 查看Nginx日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log

# 数据库备份
cp /data/www/vekus/vekus.db /backup/vekus_$(date +%Y%m%d).db

# 更新部署
cd /data/www/vekus
systemctl stop vekus-api
cp -r /path/to/new/backend ./
cp -r /path/to/new/dist ./
systemctl start vekus-api
```
