Vekus 钣金 AI 报价系统 —— 技术设计与实施方案
1. 项目概述
1.1 背景与目标
Vekus 是一个面向钣金加工行业的 AI 报价工作台，旨在帮助工厂实现从图纸上传、自动识别、参数修正、报价计算到客户管理与交易撮合的完整闭环。系统支持老板、业务员、客户三类角色，覆盖桌面端与移动端。

核心目标：

不依赖简单 OCR，而是对 DWG / 三维文件 / PDF / 图片做结构化理解

自动推断报价所需参数（板厚、折弯数、切割长度、孔数、焊接量等）

支持人工修正与二次计算，输出可直接用于报价的 quotePayload

提供报价单生成、历史查询、客户管理、交易市场、AI 客服等增值功能

1.2 用户角色
角色	能力
老板	配置报价参数与利润率、管理业务员与客户、查看经营报表、管理推荐奖励
业务员	上传图纸、触发识别、人工修正参数、生成报价、管理客户与图纸、标记成交
客户	查看报价结果、浏览交易市场、联系卖家、查看个人资料
1.3 核心主链路
业务员选择客户 → 上传图纸 → AI 识别参数 → 人工修正 → 系统自动报价 → 生成报价链接 → 导出 PDF/Excel → 分享给客户 → 跟踪成交状态并归档。

2. 系统架构
2.1 整体处理流程
text
DWG/PDF/图片/三维文件
      ↓
 文件类型分流
      ↓
┌─────────────┬─────────────┬─────────────┐
│ CAD结构解析  │  OCR+视觉   │ 3D特征提取  │
└─────────────┴─────────────┴─────────────┘
      ↓
  特征归类 & AI 推断
      ↓
  二次计算（切割长度、折弯次数、喷涂面积等）
      ↓
  输出 quotePayload + confidence
2.2 文件类型与处理策略
文件类型	处理方式
DWG / DXF	CAD 结构解析（图层、块、标注、几何）+ 规则补全
STEP / STP / X_T / SLDPRT	三维几何特征提取 → 判断是否为钣金件 → 映射为报价字段
PDF / 图片（PNG/JPG）	OCR + 视觉模型识别（标注、尺寸、技术要求）
2.3 统一 AI 识图管道
所有文件类型统一接入 unified-ai-pipeline，按类型选择前处理策略，最终输出 confidence、detections、quotePayload。处理器必须可插拔，支持 MockProcessor、ClaudeProcessor、QwenProcessor 等。

2.4 异步识图队列
为避免大文件阻塞前端，采用异步任务队列：

POST /api/ai/scan 上传文件，立即返回 scanId

后台线程（或 Celery/Redis Queue）处理识别

GET /api/ai/scan/{scanId} 查询状态（pending/processing/done/failed）

完成后返回完整 quotePayload

当前使用内存队列 + 后台线程，适合单机部署；后续可升级为持久化任务表 + 分布式队列。

5. 文件识别与提取规则
5.1 DWG / DXF 结构提取
图层：包含 DIM、尺寸 → 尺寸层；CENTER、孔 → 孔位层；BEND、折弯 → 折弯层；WELD、焊 → 焊接层。

文本：优先读取标题栏 → 技术说明 → 视图附近文字；用于推断材料、厚度、表面处理。

尺寸：线性尺寸（统一 mm）、角度尺寸、半径/直径。冲突时优先尺寸标注，次选几何计算。

块与阵列：名称含 HOLE、BOLT、WELD 分别对应孔、标准件、焊接符号；阵列数量直接参与统计。

几何实体：闭合多段线视为轮廓；圆形视为孔；SPLINE 需离散化。

5.2 三维文件识别规则（STEP / X_T / SLDPRT）
STEP/STP：提取实体、面、边界、装配层级；若存在薄板特征优先进入钣金件判断。

X_T：提取曲面边界、实体边界、孔/槽/圆角；闭合轮廓视为外形。

SLDPRT：利用特征树读取薄板特征、折弯特征、孔向导、切除拉伸。

钣金件判定条件：厚度 << 长宽；存在折弯边/折弯半径；孔、开槽、切口等。

非钣金件处理：折弯数置 0，切割长度按外轮廓估算，人工确认是否进入报价流程。

5.3 二次计算规则
字段	计算方式
切割长度	外轮廓周长 + 内部切割路径长度求和
折弯总数	统计折弯线数量，多视图去重，结合剖视图验证
喷涂面积	外表面积 - 不喷涂区域，多面展开时累加可喷涂面
孔加工量	光孔、螺纹孔、沉头孔分别统计（孔标注 + 块统计 + 阵列倍增）
焊接量	圆点符号 → 点焊数；连续线性焊缝 → 满焊长度；结合引线说明
毛坯重量	体积 × 材料密度（或人工输入）
5.4 人工确认触发条件
材料 / 板厚缺失

孔类冲突（如同时出现螺纹与光孔标注）

折弯线重复或无法判定

焊接符号模糊

表面处理无法识别

置信度低于 0.80

6.2 利润与费率
客户利润率（展示给客户）

内部利润率（老板核算）
两者均可由老板在后台配置。

6.3 参数字典（pricing_parameters）
支持动态扩展，分类包括：材料、表面处理、焊接参数、折弯参数、孔类参数、网孔板参数等。

字段：category, name, value, unit, common, enabled, sort_order

报价页仅读取 enabled=1 的参数用于选项和单价。

与 system_settings（全局配置）职责分离。

6.4 报价计算公式（示例）
text
材料费 = 材料单价 × 毛坯重量
切割费 = 切割长度 × 切割单价
折弯费 = 折弯次数 × 折弯单价
孔加工费 = 光孔数×光孔单价 + 螺纹孔数×螺纹单价 + 沉头孔数×沉头单价
焊接费 = 点焊数×点焊单价 + 满焊长度×满焊单价
喷涂费 = 喷涂面积 × 表面处理单价
总成本 = 材料费 + 切割费 + 折弯费 + 孔加工费 + 焊接费 + 喷涂费 + 额外费
总价 = 总成本 × 税管费系数 × 折扣系数 + 包装费
利润 = 总价 - 总成本
