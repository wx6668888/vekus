# Vekus 项目设计规范

> 依据当前项目首页、报价页、历史页等前端页面整理，作为现有产品视觉与交互的统一规范参考。

## 1. 设计目标

Vekus 是一个面向钣金报价场景的 AI 工作台，整体设计目标是：

- 强调“专业、可信、效率高”的产品气质
- 兼顾老板、业务员、客户三类角色的使用体验
- 在手机端和电脑端保持统一体验，并优先保证移动端可用性
- 将“图纸识别、参数配置、报价生成、历史查询、分享导出”作为核心流程

## 2. 设计风格

### 2.1 整体风格

当前项目采用的是偏“现代 SaaS + 金融感 + 仪表盘”的视觉风格：

- 大面积浅色背景，配合轻微渐变
- 卡片化布局，强调层次和信息分组
- 深色 Hero 区块搭配亮色强调按钮
- 使用适度阴影、圆角和毛玻璃效果，提升精致感
- 信息密度适中，偏向高效阅读而不是娱乐化表达

### 2.2 关键词

- 专业
- 清晰
- 轻量
- 高效
- 可追踪
- 工具型

## 3. 色彩规范

### 3.1 主色体系

当前页面中最核心的颜色如下：

- 主色蓝：`#2563eb`
- 深色主背景：`#0f172a`
- 强调青绿：`#00f5d4`
- 页面背景浅灰蓝：`#eef3f9`
- 辅助深灰文本：`#64748b`
- 分割线：`#e2e8f0`

### 3.2 色彩使用原则

- **蓝色**：用于主要操作、选中态、关键数据强调
- **深色**：用于 Hero、标题区和高权重信息承载区
- **青绿**：用于“立即开始”“核心动作”类按钮，形成明确行动引导
- **浅灰蓝背景**：用于营造轻盈的工作台氛围
- **灰色文本**：用于辅助说明、元信息、次级标签

### 3.3 状态色建议

虽然当前页面中未形成完整状态体系，但建议保持如下语义：

- 成功：绿色系
- 警告：橙黄色系
- 错误：红色系
- 信息：蓝色系

## 4. 字体与排版

### 4.1 字体

当前项目使用系统字体栈：

- `system-ui`
- `-apple-system`
- `BlinkMacSystemFont`
- `PingFang SC`
- `Microsoft YaHei`
- `sans-serif`

### 4.2 排版原则

- 中文界面优先使用无衬线字体，保证清晰易读
- 标题使用较大字号和紧凑行高，突出层级
- 说明文字使用较小字号和较高行高，降低视觉压力
- 数字型数据使用更醒目的字号和字重，增强仪表盘感

### 4.3 常见字号层级

- 页面主标题：38px - 56px
- 区块标题：16px - 18px
- 正文说明：12px - 16px
- 标签与辅助信息：12px
- 数据值：22px - 32px

## 5. 布局规范

### 5.1 页面结构

典型页面结构如下：

- 顶部导航栏 `topbar`
- 主视觉区 `hero`
- 功能卡片区 `card / feature / board`
- 底部说明区 `footer`
- 移动端固定底部导航 `mobileNav`

### 5.2 容器宽度

当前页面采用居中大容器模式：

- 首页：`max-width: 1240px`
- 报价页：`max-width: 1600px`
- 历史页：`max-width: 1400px`

说明：

- 首页偏品牌展示和入口聚合
- 报价页偏工作台操作，容器更宽
- 历史页介于展示与查询之间

### 5.3 布局方式

主要使用 CSS Grid：

- 首页 Hero 为左右双栏
- 首页功能区为四栏卡片
- 报价页为左右双栏工作台
- 历史页为左视觉右数据的双栏结构
- 移动端统一折叠为单栏

## 6. 圆角、阴影与层次

### 6.1 圆角

当前项目的圆角策略较统一：

- 按钮：14px - 16px
- 卡片：18px - 28px
- 顶栏：20px - 22px
- 徽章：999px 胶囊形

### 6.2 阴影

阴影使用较柔和，避免厚重：

- 卡片阴影偏浅，营造悬浮感
- Hero 区块阴影更强，强调首屏重点
- 导航和浮层使用低透明度阴影，保持轻盈

### 6.3 分层原则

- 页面最重要的内容放在 Hero 或主卡片中
- 次级信息放在辅助卡片中
- 纯展示性信息使用浅色背景和低对比度文本

## 7. 组件规范

### 7.1 顶部导航

特点：

- 左侧品牌区，右侧页面导航
- 当前页高亮
- 桌面端使用胶囊按钮式导航
- 移动端使用固定底部导航

### 7.2 按钮

按钮分为两类：

- **主按钮**：蓝底白字或青绿色底深字，用于核心操作
- **次按钮**：白底深字，带边框，用于辅助入口

按钮要求：

- 最小高度 48px
- 圆角一致
- 字重偏粗，保证可点性
- hover 时有轻微位移反馈

### 7.3 卡片

卡片是项目最重要的承载组件：

- 白底或半透明白底
- 边框使用浅灰蓝
- 内边距统一在 18px - 20px 之间
- 适合承载表单、统计、说明、列表等内容

### 7.4 表单控件

表单控件统一要求：

- 输入框、下拉框、文本域均采用 14px 圆角
- 默认背景浅色，边框浅灰
- 最小高度 48px，适合移动端触控
- 标签采用小字号、强字重、弱颜色
- 表单密度较高，但要保持分组清晰

### 7.5 徽章与标签

用于状态、分类和快速识别：

- 胶囊形圆角
- 颜色以蓝色系为主
- 字号小，字重高
- 不承载太多信息，只做快速识别

## 8. 交互规范

### 8.1 响应式原则

项目明确采用“手机与电脑统一体验”：

- 大屏：双栏或多栏信息布局
- 中屏：自动收缩为单栏或双栏
- 小屏：强制单栏，底部固定导航

### 8.2 交互反馈

建议保持以下反馈方式：

- 按钮 hover 轻微上移
- 当前导航高亮
- 表单变化即时重新计算
- 关键数值即时更新
- 搜索与筛选即时刷新列表

### 8.3 工作流导向

页面交互应该围绕以下业务链路展开：

1. 进入报价
2. 上传图纸
3. 自动识图/手动修正
4. 计算报价
5. 生成记录
6. 查看历史
7. 分享/导出

## 9. 移动端规范

### 9.1 移动端优先事项

- 底部导航固定，便于单手操作
- 关键按钮不小于 48px
- 卡片和表单纵向堆叠，避免横向挤压
- 避免过小字体
- 避免过于复杂的多栏布局

### 9.2 安全区处理

页面已考虑 `viewport-fit=cover` 与 `safe-area-inset-bottom`，说明移动端应：

- 保留底部安全区
- 固定导航避免遮挡系统手势区
- 内容底部留足空间，防止被导航覆盖

## 10. 页面功能介绍与具体实现方式

本项目当前已有页面主要包括：`首页`、`报价页`、`历史页`、`管理后台`。以下从“页面功能”和“实现方式”两个角度详细说明。

### 10.1 首页 `index.html`

#### 页面功能

首页的作用不是直接完成报价，而是承担产品入口、能力展示和动作引导：

- 展示产品定位：Vekus 钣金报价工作台
- 说明产品面向角色：老板、业务员、客户
- 展示核心能力：图纸识别、规则报价、客户归档、导出分享
- 提供主要入口：进入统一报价、查看历史
- 通过统计卡片强化“工作台”感知

#### 具体实现方式

首页使用的是“首屏营销 + 工作台概览”的双栏布局，核心实现如下：

1. **顶部导航**
   - 使用 `.topbar` 作为固定式顶部容器
   - 左侧品牌区显示 Logo、标题、副标题
   - 右侧通过 `.chips` 渲染页面导航
   - 移动端使用 `.mobileNav` 固定底部导航，提升单手操作效率

2. **首屏 Hero 区**
   - 使用 `.panel` 作为深色信息主视觉区
   - 通过大标题、说明文案、主按钮引导用户进入报价流程
   - `.meta` 用于展示关键业务标签，例如利润率、归档、查询次数等

3. **统计卡片区**
   - 使用 `.statgrid` + `.card` 展示“今日报价、待识别图纸、成交客户、免费查询”等关键指标
   - 指标值采用大字号 `.value`，提升仪表盘感
   - 这部分适合未来替换成真实接口数据

4. **功能说明区**
   - 使用 `.featuregrid` 以四栏卡片展示四个核心能力
   - 每个功能块都包括图标、标题、说明，属于“产品能力介绍”而非操作表单

5. **快速入口区**
   - 使用 `.board` 和 `.row` 组织快捷导航
   - 入口直接链接到 `quote.html` 和 `history.html`
   - 适合新用户快速进入主流程

6. **脚本实现**
   - 通过 `window.VekusData.renderUnifiedNav('home')` 动态渲染统一导航
   - 通过 `/api/dashboard/summary` 拉取首页统计数据，失败时兜底为默认值
   - 调用 `window.VekusData.initCustomerServiceWidget()` 初始化客服或辅助组件
   - 监听 `resize` 事件，保证导航在移动端与桌面端切换时保持一致

#### 页面特点

- 首页更强调品牌感与产品价值，而不是复杂操作
- 首屏深色 Hero 形成视觉记忆点
- 布局以展示为主，操作入口为辅

### 10.2 报价页 `quote.html`

#### 页面功能

报价页是整个系统的核心操作页面，负责完整报价链路：

- 填写客户基础信息
- 录入材料、表面处理、工艺参数
- 上传图纸并触发识别
- 回填 AI 识图结果
- 手工修正识图数据
- 实时计算报价结果
- 生成客户、图纸、报价记录并跳转结果页

#### 具体实现方式

报价页是典型的“单页工作台”，通过输入即计算的方式完成报价：

1. **布局结构**
   - `layout` 使用双栏网格：左侧表单操作，右侧结果展示
   - 左侧 `.card` 放置主表单，右侧 `.card` 展示统计结果与明细
   - 适合高密度操作场景

2. **基础输入区**
   - 客户名称、数量、材料、表面处理、备注等字段直接由 HTML 表单完成
   - 字段都使用统一高度、圆角和浅色背景，便于移动端触控

3. **图纸上传与识别**
   - 通过 `<input type="file">` 接收图纸文件
   - 点击“上传并识别”会执行 `applyScan()`
   - `applyScan()` 内部会将文件封装到 `FormData` 并请求 `/api/ai/scan`
   - 接口失败时不会阻塞页面，仍会继续执行 `recalc()`，保证前端可用

4. **识图参数与手工修正区**
   - 识图结果和人工修正项分开放置
   - 示例字段包括：板厚、展开长宽、切割长度、折弯数、喷涂面积、焊点数、光孔数、螺纹孔数、沉孔数、特殊另加费用等
   - 这符合真实报价场景中“AI 给结果，人工做校正”的工作方式

5. **实时重新计算**
   - 通过给所有关键输入绑定 `input` 和 `change` 事件
   - 任意字段变化时都会调用 `recalc()`
   - `recalc()` 会把表单值组装成参数对象，并调用 `window.VekusQuoteCalculator.calculateQuote(p)`
   - 计算结果用于刷新统计卡片、报价总价、Hero 数字和明细列表

6. **报价草稿构建**
   - `recalc()` 中会同步写入 `window.__quoteDraft`
   - 草稿对象中包含：基础信息、AI 识图结果、人工修正、成本明细、利润率、单价、总价等
   - 这样在后续提交时可以直接复用，不必再重新读取表单

7. **提交报价**
   - 点击“生成报价”会执行异步提交逻辑
   - 先调用 `/api/customers` 创建或写入客户
   - 再调用 `/api/drawings` 记录图纸
   - 再调用 `/api/quotes` 生成正式报价单
   - 最后将结果写入 `localStorage` 的 `vekus_last_quote`，并跳转到 `result.html`

8. **结果展示**
   - 右侧 `.stats` 显示总成本、总价、单价、利润
   - `#heroPrice` 以大字号突出最终报价
   - `#details` 展示板厚、展开尺寸、切割长度、折弯数等关键明细

#### 页面特点

- 报价页是“实时计算型”页面，不是传统提交表单
- 任何字段变化都应立即影响结果，减少重复点击
- 当前设计适合后续继续接入更复杂的成本模型

### 10.3 历史页 `history.html`

#### 页面功能

历史页负责查询、筛选、复用报价记录：

- 查看近期开奖和历史报价概览
- 按报价单号、客户、材料、状态搜索
- 按状态切换数据集合
- 快速进入新建报价或参数设置
- 为后续“复用报价”“再次报价”提供入口

#### 具体实现方式

1. **顶部概览区**
   - `hero` 区使用深色面板承载标题和核心动作按钮
   - `toolbar` 提供“新建报价”“参数设置”两个快捷操作
   - 适合作为历史数据的入口页

2. **统计卡片区**
   - 使用 `.stats` 展示本周报价、已成交、待跟进等摘要数据
   - 当前数值是静态示例，后续可替换为真实接口聚合结果

3. **搜索功能**
   - 搜索框通过 `#search` 接收关键字
   - 监听 `input` 事件，实现实时过滤
   - 搜索字段包含：报价单号、客户、材料、状态

4. **状态筛选**
   - `statuses` 数组定义筛选项：全部、已报价、待确认、已成交
   - `renderTabs()` 根据 `active` 状态渲染按钮样式
   - 点击按钮后更新 `active` 并重新渲染列表

5. **历史列表渲染**
   - 数据来源于 `window.VekusData.loadQuoteData()`
   - 使用 `data.history` 作为报价历史数组
   - 过滤后通过模板字符串渲染 `.row` 列表
   - 每条记录显示报价单号、客户、材料、数量、金额和状态标签

6. **响应式导航**
   - `renderNav()` 根据 `matchMedia('(max-width: 720px)')` 判断当前设备
   - 桌面端渲染顶部导航，移动端渲染底部导航
   - 同时调整 `body` 的底部留白，避免列表被遮挡

#### 页面特点

- 历史页的重点是“查得快、看得清、可复用”
- 结构简单，但需要搜索和筛选实时响应
- 适合继续扩展“详情查看”“再次报价”“导出记录”等能力

### 10.4 报价结果页 `result.html`

#### 页面功能

报价结果页负责承接报价页提交后的最终结果展示：

- 展示最终报价金额
- 展示客户名、报价单号、成本明细
- 一键复制报价摘要
- 引导继续报价或查看历史

#### 具体实现方式

1. **结果承载区**
   - 使用大字号 `#totalPrice` 直接突出最终金额
   - `#meta` 用于展示客户名称和报价单号
   - 页面数据从 `localStorage` 中的 `vekus_last_quote` 读取，保证报价页与结果页之间的连续性

2. **明细展示区**
   - `#detail` 将总成本、总价、单价、利润拆分成多行展示
   - 每行使用 `.row` 和 `.badge`，让关键数字更醒目

3. **复制摘要按钮**
   - `copyBtn` 调用 `navigator.clipboard.writeText()` 复制报价摘要
   - 若浏览器不支持剪贴板，则回退到 `alert(text)` 显示文本
   - 该设计方便业务员快速发给客户或内部转发

4. **导航与继续操作**
   - 页面顶部导航通过 `window.VekusData.getDesktopNav('quote')` 和 `getMobileNav('quote')` 保持统一
   - 提供“继续报价”“查看历史”两个后续入口，形成闭环

#### 页面特点

- 结果页是报价链路的收口页
- 所有核心数据都来自报价页缓存，不依赖用户再次录入
- 适合后续继续增加分享、导出、发送微信/邮件等能力

### 10.5 参数设置页 `settings.html`

#### 页面功能

参数设置页负责维护报价规则相关的基础数据：

- 新增材料、表面处理、焊接、孔类、折弯等参数
- 按分类浏览和筛选参数
- 维护单价、说明、常用/自定义状态
- 为报价页提供统一参数来源

#### 具体实现方式

1. **分类与筛选**
   - 页面用 `categoryOrder` 定义分类顺序
   - `renderCats()` 渲染分类筛选按钮
   - `active` 控制当前分类过滤结果

2. **参数新增表单**
   - 左侧表单包含分类、名称、说明、单价四个字段
   - `addBtn` 点击后调用 `/api/settings` 保存参数
   - 提交内容包含 `category`、`name`、`meta`、`price`、`common`、`enabled`、`sort_order`

3. **参数列表渲染**
   - 右侧 `#paramList` 根据搜索和分类条件渲染参数
   - 每条参数使用 `.row` 展示名称、分类、说明和价格
   - 通过 `.badge` 区分“常用”和“自定义”

4. **加载与初始化**
   - 通过 `window.VekusData.loadParameterCatalog()` 加载参数目录
   - 加载失败时仍允许页面显示空列表，保证基础可用

#### 页面特点

- 参数设置页是报价规则的底层配置入口
- 数据结构应保持稳定，便于报价计算复用
- 后续适合增加编辑、启停、排序、批量导入等能力

### 10.6 交易市场页 `marketplace.html`

#### 页面功能

交易市场页用于用户之间的库存设备、成品、半成品和产品出售：

- 发布库存设备、闲置物料、成品和半成品
- 浏览其他用户的出售或转让信息
- 按价格、区域、类型等维度查看交易信息
- 为后续聊天、求购、收藏、撮合提供基础入口

#### 具体实现方式

1. **列表展示**
   - 页面主体是 `#list` 列表容器
   - 数据由 `window.VekusData.loadMarketplaceListings()` 提供
   - 每条记录使用 `.row` 展示标题、品类、地区、描述和价格

2. **字段映射**
   - 标题优先使用 `title`
   - 类型优先使用 `product_type`
   - 地区优先使用 `region`
   - 描述字段兼容 `description` 和 `desc`
   - 价格通过 `window.VekusData.money()` 格式化显示

3. **导航与客服**
   - 同步渲染桌面端和移动端导航
   - 初始化客服组件，便于后续补充咨询入口

#### 页面特点

- 交易模块的业务边界应该明确为“用户之间的库存设备与产品出售/转让”
- 当前页面更像交易列表展示层，后续可继续扩展下单、联系、聊天等能力
- 适合与客户管理、个人中心联动

### 10.7 个人中心页 `profile.html`

#### 页面功能

个人中心页用于展示当前登录用户信息和账号操作：

- 显示用户名、角色和工厂信息
- 作为当前登录账号的确认入口
- 支持切换账号
- 为后续个人资料、收藏、消息、设置提供入口

#### 具体实现方式

1. **用户信息展示**
   - 从 `localStorage` 中读取 `vekus_profile`
   - 优先使用 `display_name`、`username`、`name` 展示用户名
   - 角色信息组合 `role` 和 `factory_name`

2. **账号操作**
   - 提供“切换账号”入口，跳转到 `login.html`
   - 页面文案中预留“退出登录”语义，后续可补真正清除会话逻辑

3. **导航联动**
   - 使用统一导航渲染 `profile` 当前状态
   - 保持和首页、报价页一致的站点导航体验

#### 页面特点

- 个人中心页是身份体系的一部分
- 适合与权限、消息、收藏和账号设置功能联动
- 当前实现很轻，可继续扩展为完整用户中心

### 10.8 登录页 `login.html`

#### 页面功能

登录页是系统入口，负责完成身份验证和角色跳转：

- 支持手机号 + 密码登录
- 支持微信扫码登录预留
- 支持小程序直接登录的入口概念
- 登录成功后按角色跳转到不同工作台

#### 具体实现方式

1. **登录方式切换**
   - 使用 `tabPhone` 和 `tabWechat` 两个标签切换登录方式
   - `setTab()` 控制 `phonePane` 与 `wechatPane` 显隐

2. **手机号登录**
   - 点击登录按钮后请求 `/api/auth/login`
   - 成功后把 `token` 存入 `localStorage` 的 `vekus_auth_token`
   - 同时把用户信息存入 `vekus_profile`
   - 根据 `role` 执行不同跳转：老板去首页，业务员去报价页，其他角色去个人中心页

3. **会话恢复**
   - 页面初始化时尝试请求 `/api/auth/session`
   - 如果本地 token 有效，会直接跳转到对应工作台

4. **微信扫码占位**
   - 页面预留二维码区域和说明文案
   - 适合作为后续微信开放平台接入的位置

#### 页面特点

- 登录页是系统权限和入口控制的关键页面
- 需要和角色、会话、路由跳转强关联
- 适合作为后续真实账号体系的基础

### 10.9 管理后台 `admin-dashboard.html`

#### 页面功能

管理后台主要给老板或管理员使用，负责全局管理与决策支持：

- 查看报价总览
- 查看客户和待处理任务
- 监控成交状态和业务员绩效
- 进入客户管理和参数配置页面
- 快速了解系统运行状态

#### 具体实现方式

1. **总览 Hero 区**
   - 使用深色 `panel` 作为管理后台的视觉主入口
   - 标题强调“统一管理报价、客户、图纸和成交状态”
   - 提供“进入客户管理”“新建报价”按钮，形成闭环

2. **指标卡片**
   - 右侧 `.metrics` 展示今日报价、已成交、待跟进客户、推荐查询等关键数据
   - 这些卡片适合连接统计接口后直接展示实时数据

3. **趋势与概览模块**
   - 使用 `.split` 和 `.mini-list` 分两列展示系统概览
   - 当前内容偏静态说明，可作为未来仪表盘趋势图的替代位

4. **任务队列**
   - `.list` + `.row` 用于展示待处理图纸、待确认报价、待标记成交、推荐待发放等任务
   - 每个任务右侧用 `.badge` 显示数量
   - 这类设计适合后续接入待办中心

5. **最近报价单表格**
   - 使用原生 `<table>` 展示最近报价单
   - 表头固定、宽表横向滚动，适合后台密集信息
   - 通过 `loadQuoteData()` 拉取 `data.history`，仅取前 5 条进行展示
   - 状态使用颜色区分：成交、待确认、异常/失败等

6. **业务员绩效模块**
   - 右侧列表展示业务员排行、报价数、成交数、复用率
   - 适合做成老板视角的快速比较面板

#### 页面特点

- 管理后台更强调“全局控制”和“任务监控”
- 信息密度高于前台页面，但仍保持卡片化和响应式
- 后续可扩展为客户管理、权限管理、参数管理的总入口

## 11. 页面输入 / 输出 / 状态流转

这一部分用于明确每个页面“接收什么、产出什么、状态如何变化”，便于前后端协作。

### 11.1 首页

- **输入**：首页统计接口数据、统一导航状态、客服组件状态
- **输出**：页面入口跳转、产品介绍、关键数据概览
- **状态流转**：默认展示 → 拉取统计数据 → 失败兜底 → 保持可浏览状态

### 11.2 报价页

- **输入**：客户名称、数量、材料价格、表面处理价格、识图参数、人工修正项、税管费系数、折扣系数、包装费、图纸文件
- **输出**：报价草稿 `window.__quoteDraft`、客户记录、图纸记录、报价单记录、`vekus_last_quote` 本地缓存、结果页跳转
- **状态流转**：基础填写 → 上传识图 → 参数修正 → 实时计算 → 生成草稿 → 提交入库 → 跳转结果页

### 11.3 结果页

- **输入**：`localStorage` 中的 `vekus_last_quote`
- **输出**：最终报价展示、报价摘要复制文本、继续报价跳转、历史页跳转
- **状态流转**：读取缓存 → 渲染总价与明细 → 可复制摘要 → 进入下一步操作

### 11.4 历史页

- **输入**：报价历史数据、搜索关键字、状态筛选条件、页面宽度变化
- **输出**：过滤后的历史列表、页面跳转入口、当前筛选状态
- **状态流转**：默认加载 → 渲染全部列表 → 搜索 / 筛选 → 列表重新渲染 → 进入报价复用

### 11.5 参数设置页

- **输入**：参数分类、名称、说明、价格、搜索关键字、分类筛选
- **输出**：新增参数记录、参数列表、筛选后的目录结果
- **状态流转**：加载目录 → 分类筛选 → 搜索过滤 → 新增保存 → 刷新目录

### 11.6 交易市场页

- **输入**：交易列表数据
- **输出**：库存设备 / 产品出售列表、价格展示、交易信息入口
- **状态流转**：加载列表 → 渲染商品 → 可继续扩展筛选 / 联系 / 详情

### 11.7 个人中心页

- **输入**：本地用户资料 `vekus_profile`
- **输出**：用户信息展示、切换账号入口
- **状态流转**：读取本地资料 → 渲染身份信息 → 进入登录或后续设置

### 11.8 登录页

- **输入**：手机号、密码、登录方式、会话 token
- **输出**：`vekus_auth_token`、`vekus_profile`、角色跳转结果
- **状态流转**：输入验证 → 提交登录 → 保存会话 → 按角色跳转 → 会话恢复

### 11.9 管理后台

- **输入**：历史报价数据、客户管理入口、后台统计数据
- **输出**：总览指标、任务队列、报价表格、绩效排名、客户管理跳转
- **状态流转**：加载后台概览 → 拉取数据 → 填充统计与表格 → 持续查看与跳转

## 12. 全局组件规范

### 12.1 导航组件

- **桌面端顶部导航**：用于站点级页面切换，常见于首页、报价页、历史页、后台
- **移动端底部导航**：用于主要功能入口，优先保留首页、报价、交易、我的等高频入口
- **激活态**：当前页面必须高亮，防止用户迷失

### 12.2 卡片组件

- 用于承载统计、表单、列表、说明、任务队列
- 白底卡片用于信息展示，深色卡片用于核心视觉
- 卡片之间保持统一间距，避免拥挤

### 12.3 表单组件

- 输入框、下拉框、文本域、文件上传统一使用浅色背景和圆角边框
- 表单最小高度建议保持 48px
- 输入变化应尽量实时反馈，尤其是报价页

### 12.4 按钮组件

- 主按钮用于核心动作，如“进入统一报价”“生成报价”“新建报价”
- 次按钮用于辅助操作，如“查看历史”“参数设置”
- 高度、圆角、字重需保持一致

### 12.5 徽章与状态标签

- 用于状态识别、步骤提示、任务数量提示
- 推荐使用胶囊形样式
- 状态颜色应和语义绑定，避免随意变化

### 12.6 表格组件

- 用于后台类密集信息展示
- 默认支持横向滚动，避免在小屏上压缩内容
- 表头应清晰，状态字段应有颜色区分

### 12.7 空状态与错误态

- 列表为空时应显示可理解的提示文案
- 接口失败应保留页面骨架或默认数据
- 不建议因单个接口失败而阻断整页展示

## 13. 接口与字段约定

这一部分建议作为前后端对接基线，保证页面实现一致。

### 13.1 首页接口

- `GET /api/dashboard/summary`
- **用途**：拉取首页统计数据
- **建议字段**：`quotes`、`customers`、`drawings`、`listings`
- **兜底策略**：接口失败时显示 0 或本地默认值

### 13.2 图纸识别接口

- `POST /api/ai/scan`
- **用途**：上传图纸并触发识别
- **输入**：`FormData.file`
- **输出建议**：板厚、展开尺寸、切割长度、折弯数、喷涂面积等识图结果
- **注意**：当前前端已经把请求结果视为“可选增强”，失败不能中断报价页

### 13.3 客户接口

- `POST /api/customers`
- **用途**：创建或写入客户资料
- **建议字段**：`customerCode`、`name`、`contactName`、`assignedSalesId`、`assignedBossId`、`dealStatus`、`tags`

### 13.4 图纸接口

- `POST /api/drawings`
- **用途**：保存图纸记录
- **建议字段**：`ownerUserId`、`customerId`、`fileName`、`fileType`、`drawingNo`、`status`

### 13.5 报价接口

- `POST /api/quotes`
- **用途**：保存报价单
- **建议字段**：报价编号、客户 ID、业务员 ID、老板 ID、材料信息、识图结果、成本、单价、总价、利润率、备注等
- **说明**：报价页中已直接把草稿和表单值组装后提交

### 13.6 历史数据加载

- `window.VekusData.loadQuoteData()`
- **用途**：加载历史报价与示例数据
- **输出**：包含 `history` 的数据对象
- **前端用途**：历史页和管理后台共用

## 14. 页面跳转关系

建议将当前系统的页面流转理解为以下链路：

- `首页` → `报价页`
- `首页` → `历史页`
- `报价页` → `结果页`
- `结果页` → `历史页`
- `历史页` → `报价页`（复用报价）
- `管理后台` → `客户管理` / `参数设置`

这个跳转关系应该保持稳定，避免用户在多个页面之间迷路。

## 15. 文案风格

当前项目文案偏业务工具风格，建议保持：

- 简洁直接
- 避免过度营销化
- 使用动作导向表达，如“进入统一报价”“查看历史”“参数设置”

- 统一卡片圆角、阴影、边框颜色
- 统一按钮高度与字体粗细
- 统一导航系统，桌面端顶部、移动端底部
- 统一数字展示风格，突出报价类数据

## 17. 推荐断点

当前页面主要使用以下断点：

- `980px` 附近：从多栏转为单栏
- `900px` 附近：部分模块折叠

## 18. 权限与角色说明

当前项目可以按角色划分为老板、业务员、客户、管理员四类。虽然前端页面中有些角色还没有完全做成强权限控制，但文档层面建议明确边界，后续开发时按此标准实现。

### 18.1 老板

- **核心目标**：查看全局经营状况、配置报价规则、管理客户资产
- **可访问页面**：首页、历史页、参数设置页、管理后台、客户管理页
- **主要能力**：
  - 查看报价总览和成交情况
  - 调整材料、工艺、利润率相关参数
  - 查看客户归属和业务员绩效
  - 监控库存交易与平台数据

### 18.2 业务员

- **核心目标**：录入客户、上传图纸、生成报价、持续跟进
- **可访问页面**：报价页、结果页、历史页、个人中心页、交易市场页
- **主要能力**：
  - 新建报价并触发图纸识别
  - 查看历史报价并复用
  - 通过结果页复制报价摘要给客户
  - 在客户管理中查看归属和跟进状态

### 18.3 客户

- **核心目标**：查看报价结果、了解产品信息、联系业务人员
- **可访问页面**：结果页、交易市场页、个人中心页、登录页
- **主要能力**：
  - 查看最终报价或报价分享页
  - 浏览库存设备和产品出售内容
  - 通过平台与业务员或卖家沟通

### 18.4 管理员

- **核心目标**：维护平台数据和系统配置
- **可访问页面**：管理后台、客户管理页、参数设置页、登录页
- **主要能力**：
  - 管理报价、客户、图纸、参数和交易数据
  - 处理系统级配置与基础账号信息
  - 查看全站统计和任务队列

### 18.5 权限控制建议

后续实现时建议遵循以下原则：

- **页面级权限**：不同角色进入不同首页或工作台
- **按钮级权限**：如“新增参数”“删除客户”“成交标记”等需要角色控制
- **数据级权限**：业务员只能看到自己的客户和报价，老板可看全局
- **兜底策略**：无权限时展示说明页或跳转登录页，不要直接报错

## 19. 结论

Vekus 当前项目的设计规范可以概括为：

- **现代化企业工具风格**
- **浅色背景 + 深色 Hero + 蓝色主操作**
- **卡片化信息组织**
- **移动端与桌面端统一**
- **围绕报价业务流程构建界面**
- **交易模块明确面向用户之间的库存设备和产品出售**

这套风格适合继续扩展到后续的客户管理、交易、参数设置、个人中心等页面，建议保持一致性，避免引入风格过多变化。
