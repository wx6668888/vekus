# Vekus 技术文档

> 本文档用于说明当前项目的前端页面结构、页面职责、数据流、接口约定、核心脚本逻辑和可扩展方向，便于前后端协作与后续迭代。

## 1. 项目概览

Vekus 是一个面向钣金报价场景的 Web 工作台，包含以下核心能力：

- 图纸上传与识别
- 报价参数配置与实时计算
- 报价结果展示与复制分享
- 历史报价查询与复用
- 客户管理与成交状态管理
- 参数维护与报价规则管理
- 用户交易市场（库存设备 / 产品出售）
- 登录、个人中心与角色跳转
- 管理后台总览

项目采用纯前端页面 + 统一数据脚本 + 后端 API 的结构，页面间通过 `localStorage`、统一导航和 API 进行协作。

## 4. 前端运行方式

### 4.1 静态页面加载

页面直接通过浏览器访问 HTML 文件即可运行，主要逻辑由内嵌脚本和公共 JS 文件提供。

### 4.2 页面之间的衔接

- 首页跳转到报价页、历史页、交易页、个人中心
- 报价页生成结果后跳转结果页
- 结果页可跳转历史页或继续报价
- 登录页成功后按角色跳转
- 后台跳转客户管理和参数设置

### 4.3 状态存储

页面状态主要分为三种：

1. **短期页面状态**
   - 当前筛选项
   - 当前输入值
   - 当前选中标签

2. **跨页面临时状态**
   - `vekus_last_quote`
   - `vekus_auth_token`
   - `vekus_profile`

3. **业务数据状态**
   - 客户
   - 图纸
   - 报价单
   - 参数目录
   - 交易列表

## 5. 核心数据流

## 5.1 报价流程数据流

报价流程是系统最核心的数据链路：

1. 用户进入 `quote.html`
2. 填写客户、材料、工艺、识图参数
3. 上传图纸，调用 `/api/ai/scan`
4. 前端重新计算报价
5. 点击生成报价，依次写入客户、图纸、报价单
6. 将报价结果写入 `localStorage`
7. 跳转到 `result.html`
8. 用户可复制摘要或进入历史页

### 5.2 历史查询数据流

1. 页面调用 `window.VekusData.loadQuoteData()`
2. 读取 `history` 数据
3. 按搜索词和状态过滤
4. 渲染列表或表格
5. 点击记录回到报价复用入口

### 5.3 参数配置数据流

1. 页面加载参数目录
2. 用户按分类筛选或搜索
3. 可新增参数记录
4. 调用 `/api/settings`
5. 重新加载目录

### 5.4 登录数据流

1. 用户输入手机号和密码
2. 调用 `/api/auth/login`
3. 成功后保存 token 和 user profile
4. 页面按角色跳转
5. 下次访问时调用 `/api/auth/session` 恢复会话

## 6. 页面级技术说明

### 6.1 首页 `frontend-pages/index.html`

#### 6.1.1 作用

首页主要承担品牌展示、产品定位说明和功能入口聚合。

#### 6.1.2 关键结构

- 顶部品牌区：Logo、标题、副标题
- 桌面导航：页面入口
- 移动底部导航：高频功能入口
- Hero 区：产品价值、主 CTA
- 统计卡片区：展示关键业务指标
- 功能卡片区：展示产品能力
- 快速入口区：进入报价和历史

#### 6.1.3 主要脚本

- `window.VekusData.renderUnifiedNav('home')`
- `fetch('/api/dashboard/summary')`
- `window.VekusData.initCustomerServiceWidget()`

#### 6.1.4 重点技术点

- 首屏使用双栏 Grid
- 小屏切换为单栏
- 导航组件在 resize 时重新渲染
- 统计区具备接口失败兜底能力

### 6.2 报价页 `frontend-pages/quote.html`

#### 6.2.1 作用

报价页是系统最核心的业务页面，负责图纸识别、参数输入、成本计算和报价入库。

#### 6.2.2 页面结构

- 左侧：客户信息、工艺参数、图纸上传、手工修正、提交按钮
- 右侧：报价统计、最终价格、明细列表

#### 6.2.3 核心逻辑

1. **表单输入监听**
   - 所有字段绑定 `input` 和 `change`
   - 任一字段变化都会触发 `recalc()`

2. **报价计算**
   - 通过 `window.VekusQuoteCalculator.calculateQuote(p)` 计算
   - 计算参数包含数量、材料价格、表面价格、切割长度、折弯数、喷涂面积、焊点数等

3. **草稿构建**
   - 计算结果同步写入 `window.__quoteDraft`
   - 草稿包含基础信息、AI 识图数据、人工修正、成本明细、利润率、单价、总价

4. **图纸识别**
   - 点击上传识别后调用 `/api/ai/scan`
   - 使用 `FormData` 上传图纸文件
   - 即使失败也不阻断报价页继续使用

5. **报价提交**
   - 创建客户 `/api/customers`
   - 创建图纸 `/api/drawings`
   - 创建报价 `/api/quotes`
   - 将结果写入 `localStorage`
   - 跳转 `result.html`

#### 6.2.4 技术要点

- 实时计算型页面，不是传统提交表单
- 结果区必须随输入即时变化
- 任何单个接口失败都不应阻塞主流程

### 6.3 报价结果页 `frontend-pages/result.html`

#### 6.3.1 作用

结果页用于展示最终报价、成本明细和摘要分享。

#### 6.3.2 数据来源

- `localStorage.vekus_last_quote`

#### 6.3.3 核心逻辑

- 读取缓存报价数据
- 展示总价与报价单号
- 展示总成本、单价、利润
- 复制报价摘要到剪贴板
- 提供继续报价和查看历史入口

#### 6.3.4 技术要点

- 剪贴板 API 兼容性处理
- 无缓存时应显示兜底内容
- 页面间通过本地缓存完成临时状态传递

### 6.4 历史页 `frontend-pages/history.html`

#### 6.4.1 作用

历史页用于查询、筛选和复用报价记录。

#### 6.4.2 核心逻辑

- 调用 `loadQuoteData()` 获取历史数据
- 使用搜索框实时过滤
- 使用状态标签切换数据集
- 通过模板字符串渲染列表

#### 6.4.3 技术要点

- 本地搜索 + 状态过滤
- 响应式导航在桌面和移动端切换
- 空状态要清晰提示
- 历史记录可直接跳回报价页做复用

### 6.5 参数设置页 `frontend-pages/settings.html`

#### 6.5.1 作用

维护材料、表面处理、焊接、孔类、折弯等报价参数。

#### 6.5.2 核心逻辑

- 加载参数目录
- 分类筛选
- 关键词搜索
- 新增参数调用 `/api/settings`
- 成功后重新拉取目录

#### 6.5.3 技术要点

- 参数目录是报价规则基础
- 页面需要支持常用与自定义标识
- 空列表和接口失败都应可继续使用

### 6.6 交易市场页 `frontend-pages/marketplace.html`

#### 6.6.1 作用

交易模块用于用户之间的库存设备、产品、半成品和成品出售与转让。

#### 6.6.2 核心逻辑

- 调用 `loadMarketplaceListings()` 获取列表
- 按统一字段映射标题、品类、区域、描述和价格
- 渲染交易商品卡片列表

#### 6.6.3 技术要点

- 交易模块的业务边界必须清晰
- 当前以列表展示为主，后续可扩展为详情、聊天、收藏、求购

### 6.7 个人中心页 `frontend-pages/profile.html`

#### 6.7.1 作用

展示登录用户身份和账号信息。

#### 6.7.2 核心逻辑

- 从 `localStorage.vekus_profile` 读取用户信息
- 展示用户名、角色、工厂信息
- 提供切换账号入口

### 6.8 登录页 `frontend-pages/login.html`

#### 6.8.1 作用

作为统一登录入口，完成身份验证与角色跳转。

#### 6.8.2 核心逻辑

- 手机号 + 密码登录调用 `/api/auth/login`
- 登录成功后保存 token 和 profile
- 调用 `/api/auth/session` 恢复会话
- 根据 `role` 跳转到不同页面

#### 6.8.3 技术要点

- 微信扫码登录为预留区块
- 登录后自动跳转，减少用户手动判断

### 6.9 管理后台 `admin-dashboard.html`

#### 6.9.1 作用

提供老板视角的全局概览、绩效、任务与报价监控。

#### 6.9.2 核心逻辑

- 加载报价历史数据
- 渲染统计卡片
- 渲染最近报价表
- 展示业务员绩效

#### 6.9.3 技术要点

- 表格适合后台密集信息展示
- 横向滚动适配较宽表格
- 未来可接入真实统计接口

### 6.10 客户管理页 `admin-customers.html`

#### 6.10.1 作用

用于客户归属管理、成交状态维护和历史报价复用。

#### 6.10.2 核心逻辑

- 搜索客户名、电话、业务员、标签
- 按状态过滤客户
- 渲染客户列表与报价复用表
- 提供复用报价入口

#### 6.10.3 技术要点

- 客户管理页是客户资产管理中心
- 需要兼顾列表浏览和复用操作
- 后续可扩展详情页、跟进记录、联系人维护

## 8. 接口约定

### 8.1 `GET /api/dashboard/summary`

用途：拉取首页统计数据。

建议字段：

- `quotes`
- `customers`
- `drawings`
- `listings`

### 8.2 `POST /api/ai/scan`

用途：上传图纸并触发识别。

输入：

- `FormData.file`

输出建议：

- 板厚
- 展开尺寸
- 切割长度
- 折弯数
- 喷涂面积

### 8.3 `POST /api/customers`

用途：创建或写入客户资料。

建议字段：

- `customerCode`
- `name`
- `contactName`
- `assignedSalesId`
- `assignedBossId`
- `dealStatus`
- `tags`

### 8.4 `POST /api/drawings`

用途：保存图纸记录。

建议字段：

- `ownerUserId`
- `customerId`
- `fileName`
- `fileType`
- `drawingNo`
- `status`

### 8.5 `POST /api/quotes`

用途：保存报价单。

建议字段：

- `quoteNo`
- `customerId`
- `salesId`
- `bossId`
- `projectName`
- `partName`
- `drawingId`
- `unitPrice`
- `totalPrice`
- `taxAmount`
- `materialType`
- `materialName`
- `surfaceTreatment`
- `sheetThickness`
- `quantity`
- `cutLength`
- `bendCount`
- `sprayArea`
- `remarks`

### 8.6 `POST /api/settings`

用途：新增报价参数。

建议字段：

- `category`
- `name`
- `meta`
- `price`
- `common`
- `enabled`
- `sort_order`

### 8.7 `POST /api/auth/login`

用途：账号登录。

请求体建议：

- `phone`
- `password`

返回建议：

- `token`
- `user`

### 8.8 `GET /api/auth/session`

用途：恢复会话。

请求头建议：

- `Authorization: Bearer <token>`

### 8.9 `window.VekusData.loadQuoteData()`

用途：加载历史报价数据。

返回：

- 包含 `history` 数组的对象

### 8.10 `window.VekusData.loadParameterCatalog()`

用途：加载参数目录。

### 8.11 `window.VekusData.loadMarketplaceListings()`

用途：加载交易市场列表。

## 9. 本地存储约定

### 9.1 `vekus_last_quote`

保存报价页生成的临时结果，用于结果页展示。

### 9.2 `vekus_auth_token`

保存登录后 token，用于会话恢复。

### 9.3 `vekus_profile`

保存用户资料，用于个人中心展示和角色跳转。

## 10. 响应式策略

### 10.1 桌面端

- 顶部导航
- 双栏或多栏布局
- 表格可横向滚动
- 更高信息密度

### 10.2 移动端

- 底部固定导航
- 单栏布局
- 关键按钮高度充足
- 避免过小文字和多列拥挤

### 10.3 断点建议

- `980px`：主布局转单栏
- `900px`：次级卡片折叠
- `720px`：移动导航启用
- `640px`：进一步压缩间距和字体

## 11. 异常与兜底

### 11.1 接口失败

- 页面保留默认数据或空状态
- 不阻断主流程
- 控制台输出错误日志即可

### 11.2 本地缓存缺失

- 结果页、个人中心页要有默认值
- 不应出现空白页

### 11.3 计算失败

- 报价页应捕获异常
- 保持表单可操作
- 显示控制台错误，避免页面崩溃

### 11.4 空列表

- 历史、客户、交易、参数页都要提供空状态文案

## 12. 扩展建议

后续建议优先扩展以下方向：

- 报价结果页增加导出与分享
- 客户管理页增加详情和跟进记录
- 参数设置页增加编辑、排序、启停
- 交易市场页增加详情、联系、收藏、求购
- 管理后台接入真实统计接口和图表
- 登录页接入短信验证码或微信扫码正式流程

## 13. 总结

当前项目的技术结构适合快速迭代：

- 页面职责清晰
- 核心数据流简单直接
- 依赖统一的数据脚本和少量 API
- 通过本地缓存解决页面间状态传递
- 通过响应式布局同时覆盖桌面和移动端

如果要继续演进，建议优先统一接口字段、补齐权限控制，并把交易、客户、报价三条主链路进一步打通。
