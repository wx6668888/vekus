# Vekus 数据库补充清单与建表/改表建议

> 说明：本文件用于记录当前项目数据库结构的补充建议、字段对齐方案和后续落地优先级。每次做数据库补充或改表后，应同步更新本文档，并确保相关代码与页面已保存。

## 1. 结论摘要

当前数据库整体可支撑主链路，但仍建议优先补齐以下能力：

1. `system_settings` 继续只承担全局配置
2. 新增 `pricing_parameters` 承担报价参数字典
3. `quotes` 需要补版本、有效期、利润、导出与分享字段
4. `drawings` 需要补 AI 识图追踪字段
5. `customers` 需要补生命周期字段
6. `users` 需要补登录、微信、头像、软删除字段
7. 建议新增 `audit_logs` 用于操作审计
8. `listings` 与 `chat_messages` 可继续增强交易侧能力

## 3. 建表/改表建议

### 3.1 新增 `pricing_parameters`
用于承载报价参数字典。

建议字段：
- `id` bigint 主键
- `category` varchar 分类，如材料、表面处理、焊接参数
- `name` varchar 参数名称
- `value` decimal 参数值或系数
- `unit` varchar 单位
- `common` tinyint 是否常用
- `enabled` tinyint 是否启用
- `sort_order` int 排序
- `remark` text 备注
- `created_at` timestamp
- `updated_at` timestamp

### 3.2 新增 `audit_logs`
用于审计和排障。

建议字段：
- `id` bigint 主键
- `user_id` bigint 操作人
- `action` varchar 动作类型
- `target_type` varchar 目标类型
- `target_id` bigint 目标ID
- `before_json` json 修改前
- `after_json` json 修改后
- `ip` varchar 客户端 IP
- `created_at` timestamp

---

## 4. 不建议立即新增的表

当前不建议立刻新建太多表，优先把下面表的字段补齐：
- `users`
- `customers`
- `drawings`
- `quotes`
- `listings`
- `chat_messages`
- `system_settings`
- `pricing_parameters`

原因：
- 现在主链路已经有表
- 先补字段比扩散表更稳
- 可以尽快让前后端和数据库一致
