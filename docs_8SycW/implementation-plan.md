# Vekus 钣金报价系统实施方案

## 1. 项目目标

将当前恢复出的 PWA 报价系统升级为一套面向钣金厂的正式业务系统，支持：

- 业务员上传客户图纸
- 系统自动识别图纸参数
- 按老板配置的单价与利润规则自动报价
- 生成可分享的报价页面链接
- 支持导出 Excel / PDF
- 支持微信 / 邮箱转发
- 支持客户、图纸、报价单、成交状态管理
- 支持推荐奖励与免费查询次数
- 同时支持手机端与电脑端

---

## 2. 用户角色

### 2.1 用户
客户侧用户，只查看报价结果，不参与后台配置。

能力：
- 打开报价链接
- 查看报价详情
- 接收分享内容
- 查看导出的 PDF / Excel

### 2.2 业务员
工厂内部实际操作人员。

能力：
- 上传图纸
- 选择或新建客户
- 触发 AI 识别
- 手工修正参数
- 生成报价
- 标记成交状态
- 管理自己负责的客户与图纸
- 一键复用历史报价

### 2.3 老板
最高权限角色，负责配置和管理。

能力：
- 管理业务员
- 管理客户归属
- 配置参数字段与单价
- 配置利润率
- 查看成交率、转化率、报价量
- 管理推荐奖励与免费查询次数
- 导出全局数据

## 6. 报价规则设计

### 6.1 必须保留现有参数
当前已有参数应全部保留，不能删除：

- 材料
- 表面处理
- 切割长度
- 折弯数
- 喷涂面积
- 焊点数
- 满焊长度
- 光孔数
- 螺纹孔数
- 沉孔数
- 额外费用
- 税费系数
- 折扣系数
- 包装费

### 6.2 老板可扩展字段
允许老板增加新的字段，例如：

- 工艺费
- 打样费
- 装配费
- 压铆费
- 拉丝费
- 镀锌附加费
- 加急费

### 6.3 双利润率
需要同时支持：

- 对客户展示的利润率
- 内部核算利润率

这两套利润率都应由老板可配置，并在报价单中保留历史记录。

---

## 7. 图纸与客户管理规则

### 图纸管理
- 图纸按老板、业务员、客户分层管理
- 每张图纸绑定上传人、客户、报价单
- 支持状态分类：待识别、已识别、已报价、已成交、已归档

### 客户管理
- 老板可查看全局客户
- 业务员仅看自己负责客户
- 下次报价可自动带出客户历史信息
- 支持是否成交标记

---

## 8. 推荐分享机制

建议采用邀请码机制：

- 每个用户拥有专属邀请码
- 新用户通过邀请码注册
- 推荐人获得免费查询次数
- 免费次数可用于上传识别、生成报价、导出文档等
- 老板可后台调整奖励规则

---

## 9. 视觉与体验方向

目标风格：

- 简约现代
- 科技感强
- 参考 v0.app 的产品质感
- 深浅分明、留白充足
- 卡片化布局
- 移动端和电脑端统一视觉语言

### 手机端
- 底部导航
- 单屏流程化操作
- 上传、识别、报价、分享尽量一气呵成

### 电脑端
- 左侧导航 + 右侧内容区
- 表格与筛选为主
- 适合老板与管理操作

## 11. Git 备份策略

当前恢复出的站点已应当作为首个稳定版本进行 Git 备份，避免再次因误删或覆盖导致丢失。

建议：
- 保留一个可回滚的基线提交
- 每次重构前先提交备份
- 大改动前单独建分支
- 重要数据和上传文件要排除在代码仓库之外
